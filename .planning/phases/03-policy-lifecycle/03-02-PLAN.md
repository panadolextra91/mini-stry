    `rollback(ctx: TenantContext, policyId: PolicyId, targetVersionId: PolicyVersionId, actorId: UserId): Promise<PolicyVersion>` — per D-33: find target version via `versionRepo.findById()`; throw `VersionNotFoundError` if missing. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rollback source version | Target version must exist and belong to same tenant |
| Audit log writes | Append-only — no update/delete exposed |
| Convex adapter boundary | Domain types ↔ Convex doc types; no Convex types leak to domain |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | Audit log modification | mitigate | AuditLogRepositoryPort exposes only create + find; no update/delete methods |
| T-03-06 | Information Disclosure | Cross-tenant rollback | mitigate | Rollback target version scoped by tenantId; cannot reference another tenant's version |
| T-03-07 | Tampering | Version number manipulation | mitigate | Version numbers computed server-side via getNextVersionNumber; never client-supplied |
| T-03-08 | Repudiation | Lifecycle events unaudited | mitigate | AuditEventSubscriber wired in constructor — events cannot bypass audit (D-35 synchronous) |
</threat_model>
