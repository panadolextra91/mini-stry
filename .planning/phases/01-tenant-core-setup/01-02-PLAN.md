# Plan 01-02: Create Convex Database Schema with Multi-Tenant Indexing

## Objective

Create the Convex database schema (`convex/schema.ts`) to register document structures for tenants and users, and implement indexes to enable logically isolated, high-performance database queries by `tenantId`.

## Verification Criteria

1. `convex/schema.ts` is successfully registered and compiles with standard Convex CLI validations.
2. Tables `tenants` and `users` are defined with their correct field models.
3. Database index on `tenantId` is declared on the `users` table.

## must_haves

1. Convex schema must include a `tenantId` field on `users` to guarantee data isolation.
2. Convex schema must support an index on `tenantId` to enforce logical query isolation rules.
