# Plan 01-02: Create Convex Database Schema with Multi-Tenant Indexing

## Objective

Create the Convex database schema (`convex/schema.ts`) registering dynamic data-driven collections for tenants, roles, users (linked via stable ID-based `roleId`), and audit logs, complete with logical indices on `tenantId` to enforce strict logical query isolation.

## Verification Criteria

1. `convex/schema.ts` is successfully registered and compiles with standard Convex validations.
2. Tables `tenants`, `roles`, `users`, and `auditLogs` are defined with their correct models.
3. Database indices on `tenantId` are declared on all tenant-specific tables.

## must_haves

1. Convex schema must include a `tenantId` field and an index `by_tenant` on all tenant-specific collections to guarantee logical isolation.
2. The user schema must link to roles via `roleId: v.string()` referencing the roles collection ID.
