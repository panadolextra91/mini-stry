# Plan 01-03: Implement Repository Ports, Convex Adapters, Services, and Unit Tests

## Objective

Develop the complete decoupled Hexagonal persistence flow: define repository ports for tenants, dynamic roles, and users, write their implementation using Convex database context, implement application services orchestrating registrations and dynamic role validations, and verify logic with a 100% test coverage Vitest suite using Mock Repository doubles.

## Verification Criteria

1. Interfaces `TenantRepositoryPort`, `RoleRepositoryPort`, and `UserRepositoryPort` exist.
2. Adapters `ConvexTenantRepository`, `ConvexRoleRepository`, and `ConvexUserRepository` correctly implement their corresponding port interfaces.
3. Application services `TenantService` and `UserService` implement core workflows (register tenant, register dynamic roles, register users, validate roles, retrieve manager reporting lines).
4. Running `npx vitest run user.service.spec.ts` executes successfully with 100% test coverage targets.

## must_haves

1. Core repository ports and services must have zero direct imports from Convex packages to prevent platform lock-in.
2. User registration must dynamically query `RoleRepositoryPort` to verify that the user's role has been registered for that tenant before allowing registration.
3. Unit tests must use Pure TS Mock repository classes to guarantee fast, deterministic tests without running external databases.
