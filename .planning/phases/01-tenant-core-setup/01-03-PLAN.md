# Plan 01-03: Implement Repository Ports, Convex Adapters, Services, and Unit Tests

## Objective

Develop the complete decoupled Hexagonal persistence flow: define repository ports, write their implementation using Convex database contexts, implement tenant/user application services orchestrating registration and reporting structures, and verify logic with a 100% test coverage Vitest suite using Mock Repository doubles.

## Verification Criteria

1. Interfaces `TenantRepositoryPort` and `UserRepositoryPort` exist.
2. Adapters `ConvexTenantRepository` and `ConvexUserRepository` correctly implement their corresponding port interfaces.
3. Application services `TenantService` and `UserService` implement core workflows (register tenant, register user, validate roles, retrieve manager reporting lines).
4. Running `npx vitest run user.service.spec.ts` executes successfully with 100% test coverage targets.

## must_haves

1. Core repository ports and services must have zero direct imports from Convex packages to prevent platform lock-in.
2. Unit tests must use Pure TS Mock repository classes to guarantee fast, deterministic tests without running external databases.
