# Plan 01-03: Implement Repository Ports, Convex Adapters, Services, and Unit Tests

## Objective

Develop the complete decoupled Hexagonal persistence flow: define repository ports for tenants, dynamic roles, and users, write their implementation using Convex database context, implement application services (separating `RoleService` and `UserService`), and verify all logical boundaries with a 100% test coverage Vitest suite using Mock Repository doubles.

## Verification Criteria

1. Interfaces `TenantRepositoryPort`, `RoleRepositoryPort`, and `UserRepositoryPort` exist.
2. Adapters `ConvexTenantRepository`, `ConvexRoleRepository`, and `ConvexUserRepository` correctly implement their corresponding port interfaces.
3. Application services `TenantService`, `RoleService`, and `UserService` are separated and implement core workflows.
4. Running `npx vitest run user.service.spec.ts` executes successfully with 100% test coverage targets.

## must_haves

1. Core repository ports and services must have zero direct imports from Convex packages to prevent platform lock-in.
2. User registration must dynamically query `RoleRepositoryPort` using the user's `roleId` to verify that the role is registered for that tenant before allowing registration.
3. Unit tests must use Pure TS Mock repository classes to guarantee fast, deterministic tests without running external databases.
