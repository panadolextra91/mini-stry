# Plan 01-01: Bootstrap TypeScript/Vitest Environment and Core Domain Models

## Objective

Set up the modular modular-monolith Hexagonal folder structure, configure the Vitest environment for unit testing, and implement the core tenant, user, dynamic role, and foundational AuditLog domain entities in Pure TypeScript.

## Verification Criteria

1. Modular directories under `src/modules/tenant/` exist and match the Hexagonal Architecture guidelines.
2. `vitest.config.ts` is configured and passes structural validation.
3. TypeScript compiler validates that `entities.ts` and `types.ts` are 100% pure TypeScript (no external framework database dependencies).

## must_haves

1. TypeScript compiler and Vitest runner must execute successfully.
2. Core domain classes `RoleEntity` and `AuditLogEntity` must be implemented, allowing dynamic, data-driven roles and auditability.
3. `UserEntity` role must be a string referring dynamically to a Role entity name, not a static enum.
