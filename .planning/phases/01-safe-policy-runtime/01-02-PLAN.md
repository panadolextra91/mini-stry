# Plan 01-02: Implement Lexer Scanner and AST Rule Parser

## Objective

Build the Lexer scanner to tokenize custom text DSL strings, and the AST Parser to compile token streams into strongly typed Abstract Syntax Trees representing conditional rule executions.

## Verification Criteria

1. `lexer.ts` successfully converts a policy text string into a clean array of token objects.
2. `parser.ts` compiles tokens into a valid `PolicyAST` object structure.
3. Edge cases like multiple spaces, variable parsing, and invalid tokens are correctly checked.

## must_haves

1. Lexer must strip whitespaces and parse standard comparative expressions (`<=`, `>`, `==`).
2. Parser must construct structured AST branches representing rules without any usage of Javascript native eval.
