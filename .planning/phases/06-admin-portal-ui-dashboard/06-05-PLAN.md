DemoContext (06-03): useDemoContext() → { tenantId, actorId }. All hooks pass these. Switching tenant/actor reloads everything reactively (D-55/56/57, D-63).

Data shapes (from src/modules barrels — read for field names): RequestEvaluation has status + trace (TraceEntry[]) on requestEvaluations.trace; ApprovalTask has approverId + state (TaskState: Pending/Approved/Rejected etc.) + stage; AuditLog has eventType + metadata + timestamp.

UI-SPEC layout/copy:
- Request Center (UI-02): raw JSON EvaluationContext intake (textarea or Monaco-lite); below/beside it a live execution-log list with decision state + step-by-step trace (mono). Primary CTA "Submit request". Empty state "No requests yet" / "Submit an EvaluationContext to run it against the active policy and watch the decision land here in real time." Invalid-JSON error: "That payload isn't valid JSON." / "Check for a trailing comma or unquoted key, then submit again."
- Inbox (UI-03): list of tasks assigned to actorId; inline Approve/Reject per row, state-driven (shown only to the assigned approver — always true here since findByApprover already scopes to actorId). Reject opens destructive confirm "Reject this approval task?" with destructive-red confirm "Reject". Approve is the primary accent CTA per row. Empty state "No tasks waiting on you" / "Approvals assigned to the active user appear here automatically...".
- Governance (UI-04): chronological audit timeline + comparative policy-version list (standard side-by-side/inline diff acceptable). Empty state "No audit activity" / "Published versions, rollbacks, and decisions for this tenant will be recorded here."
- Status semantics: muted neutral text + small dot/badge; accent only for in-progress/active; destructive-red for Rejected/Auto-Reject. Brief accent pulse on reactive row updates (D-63). Branded IDs + traces render in Geist Mono 14px.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client {tenantId, actorId} args → handlers | All three modules pass Demo Context identity; no auth (D-55) |
| EvaluationContext JSON → backend | Untrusted payload; JSON.parse'd on the client, structurally evaluated server-side, never eval'd |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-13 | Information Disclosure | inbox / request log / audit / version lists | mitigate | Every query passes tenantId (and actorId for inbox) to the 06-01/02 tenant-scoped read paths; no client-side cross-tenant filtering |
| T-06-14 | Spoofing / Elevation | approve/reject | mitigate | act() enforces actorId === approverId server-side; UI passes the selected actorId; wrong-approver acts rejected with UnauthorizedApproverError |
| T-06-15 | Tampering / XSS | rendered trace / audit / policy content | mitigate | React escapes by default; JSON/traces rendered as text in <pre>/mono, never dangerouslySetInnerHTML |
| T-06-16 | DoS | reactive list subscriptions | mitigate | List adapters (06-01/02) are bounded (take/order); subscriptions ride bounded result sets |
| T-06-17 | Spoofing (deferred) | client-supplied identity | accept | D-55 defers auth; documented accepted demo-scope risk; server-side scoping still enforced |
</threat_model>
