## Trust Boundaries

| Boundary                                     | Description                                                                                                                                                                                          |
| -------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| caller → application service (TenantContext) | `ctx.tenantId`/`ctx.actorId` are the only identity envelope; untrusted relative to the data they scope. This plan only defines the optional `actorId` carrier; enforcement lands in 05-02's `act()`. |
| subscriber → EventDispatcher                 | A subscriber throwing must not corrupt the emit loop or bubble into the publisher (`submit()`). This plan hardens that boundary (D-54).                                                              |
| memory fake read → tenant data               | Every fake read must filter by `ctx.tenantId` or it leaks cross-tenant rows.                                                                                                                         |
