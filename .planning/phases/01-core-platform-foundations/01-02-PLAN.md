<objective>
Define the Pure-TS domain layer for all three Phase 1 modules (directory, policy, audit). Create branded ID types, entity interfaces, the TenantContext first-class concept, repository port interfaces, custom error classes, and per-module barrels + READMEs. This plan delivers ZERO behavior — no service implementations, no Convex adapters, no in-memory fakes. Those land in plan 01-04. The output of this plan is the contract surface against which plan 01-03 (Convex schema) and plan 01-04 (services + adapters) build.

Purpose: Per the locked decisions (D-13 entities are plain interfaces, D-14 branded string IDs, D-15 each module owns its IDs, D-17 service surface, D-19 TenantContext-first methods), the domain layer must exist before services can be written. Defining ports up front means plan 01-04 can implement services AND in-memory fakes AND Convex adapters against the same contracts without scavenger-hunt exploration.

Output: Complete `src/modules/{directory,policy,audit}/` directory tree per RESEARCH.md "Recommended Project Structure" with all domain entities, branded IDs, repository ports, TenantContext, custom error classes, and barrel index.ts files. `npm run typecheck && npm run lint` exit 0.
