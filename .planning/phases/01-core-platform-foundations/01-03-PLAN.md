## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Storage layer -> Application services | Tenant-owned tables must enforce tenant prefixing at the index level; a missing tenantId-first index lets adapter code accidentally write tenant-spanning queries. CRITICAL boundary per CON-01 |
| Schema field set -> Convex Auth | If Phase 1 users table omits Auth-reserved fields, Phase 2 enabling Convex Auth fails because the strict schema rejects unknown fields; user data must then be migrated |
| PolicyVersion.content shape | If Phase 1 narrows content (e.g. defines it as a structured JSON object), storage is coupled to runtime concerns; Phase 2 JSON Schema cannot evolve independently |
| AuditLog.eventType registry | If Phase 1 closes eventType to an enum, every new event in every later phase requires a schema migration — violates D-16 |
